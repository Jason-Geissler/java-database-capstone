Title:
As a doctor, I want to manage my schedule and profile, so that I can provide accurate availability and be prepared for patient appointments.

Acceptance Criteria:

Doctor can log in using valid credentials and access the doctor dashboard.
Doctor can log out securely and is prevented from accessing protected pages afterward.
Doctor can view a calendar of upcoming appointments.
Doctor can mark unavailable time slots so patients can only book available hours.
Doctor can update profile details including specialization and contact information.
Doctor can view patient details for upcoming appointments.
Priority: High
Story Points: 8

Notes:

Unavailability should block new bookings for those time slots.
Profile updates should be immediately reflected in patient-facing views.
Patient data should be visible only for scheduled appointments.

Title:
As a patient, I want to browse doctors, manage my account, and book appointments, so that I can easily receive medical care.

Acceptance Criteria:

Patient can view a list of doctors without logging in.
Patient can register using an email and password to create an account.
Patient can log in and log out securely.
Patient can book an hour-long appointment with a doctor.
Patient can view a list of upcoming appointments.
Priority: High
Story Points: 8

Notes:

Email must be unique during registration.
Appointments should not overlap for the same doctor.
Only authenticated users can book or view appointments.
